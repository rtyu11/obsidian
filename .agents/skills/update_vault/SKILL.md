# Vault Updater Skill

## トリガー条件
- **「更新分をまとめて」**：新規・更新ハイライトを検出し、タグ・リンクを自動付与して1層目ノートを作成・更新する。
- **「〇〇について話したい」**：指定した本の1層目ノートをベースに会話モードに入り、気づきを引き出してノートに追記する。

---

## 【トリガー1】「更新分をまとめて」

### 1. 処理対象ファイルの検出
- `/Source/Kindle` と `/Source/OCR` 内のファイルを一覧取得する
- `/Notes/Books` 内の既存の1層目ノートを一覧取得する
- 以下を処理対象とする：
  - **新規**：1層目ノートがまだ存在しない本
  - **追記あり**：ハイライト数（`highlightsCount`）が1層目ノートの記録より増えている本
- 該当なしの場合は「新しい更新はありませんでした」と伝えて終了する

### 2. タグの自動付与
各本について以下のタグを判断して付与する：

#### ジャンルタグ（NDC日本十進分類法・2階層）
大分類と小分類を組み合わせて付与する。

| 大分類タグ | 小分類タグの例 |
|---|---|
| `#genre/哲学` | `#genre/哲学/心理学` `#genre/哲学/倫理学` `#genre/哲学/思想` |
| `#genre/歴史` | `#genre/歴史/日本史` `#genre/歴史/世界史` `#genre/歴史/伝記` |
| `#genre/社会科学` | `#genre/社会科学/経営` `#genre/社会科学/経済` `#genre/社会科学/教育` `#genre/社会科学/法律` |
| `#genre/自然科学` | `#genre/自然科学/医学` `#genre/自然科学/脳科学` `#genre/自然科学/生物学` |
| `#genre/技術` | `#genre/技術/情報工学` `#genre/技術/建築` |
| `#genre/文学` | `#genre/文学/日本文学` `#genre/文学/海外文学` `#genre/文学/エッセイ` |
| `#genre/芸術` | `#genre/芸術/デザイン` `#genre/芸術/音楽` |

#### キーワードタグ（内容から自動抽出）
ハイライト内容を読み取り、主要テーマ・概念を3〜6個タグとして付与する。
例：`#習慣` `#意思決定` `#睡眠` `#リーダーシップ` `#マインドセット` など

### 3. 1層目ノートの作成・更新

#### 保存先
`/Notes/Books/[本のタイトル].md`

#### ノートフォーマット（新規作成）
```markdown
# [本のタイトル]

著者: [[著者名]]
#genre/大分類/小分類 #キーワード1 #キーワード2 #キーワード3

---

## ハイライト

> [ハイライト原文 1]

> [ハイライト原文 2]

> [ハイライト原文 3]

---

## 自分の気づき

<!-- 会話後に追記される -->

---

## 関連ノート

- [[関連する1層目ノート（他の本）]]
- [[関連する2層目ノート（テーマまとめ）]]

<!-- highlightsCount: [件数] -->
```

**重要**：
- AI はハイライトの要約・分析・考察を**一切書かない**
- 「自分の気づき」セクションは空欄のまま残す
- ハイライトは原文をそのまま引用する

#### 追記ありの場合
- 新しく追加されたハイライトのみを `## ハイライト` セクションに追記する
- `highlightsCount` を最新の件数に更新する
- 「自分の気づき」セクションは編集しない

### 4. 関連リンクの自動付与
- 同じ著者の他の本ノートがあれば `## 関連ノート` にリンクを追記する
- 同じキーワードタグを持つ本ノートがあれば `## 関連ノート` にリンクを追記する
- 該当する2層目ノートがあれば `## 関連ノート` にリンクを追記する

### 5. 2層目ノートの確認・更新
- 今回追加したキーワードタグを持つ本が**2冊以上**存在する場合、そのテーマの2層目ノートを確認する
- 2層目ノートが存在しない場合は `/Notes/Themes/[テーマ名].md` を新規作成する
- 2層目ノートへの 新しい本のリンク追記のみ行う（内容の書き換えは禁止）

#### 2層目ノートフォーマット（新規作成時）
```markdown
# [テーマ名]（横断まとめ）

#キーワード #関連キーワード

---

## 複数の本での共通点

### [[本Aのノート]]
> [関連ハイライト原文]

### [[本Bのノート]]
> [関連ハイライト原文]

---

## 自分の発見

<!-- 会話後に追記される -->

---

## 参照ノート

- [[本Aのノート]]
- [[本Bのノート]]
```

**重要**：AI は「自分の発見」セクションを空欄のまま残す

### 6. 完了報告
- 作成・更新した1層目ノート一覧
- 作成・更新した2層目ノート一覧
- 新たにつながった本の組み合わせ（あれば）
- 「〇〇について話したいで会話を始められます」と案内する

---

## 【トリガー2】「〇〇について話したい」

### 1. 対象ノートの読み込み
- 指定された本の1層目ノート（`/Notes/Books/[本のタイトル].md`）を読み込む
- ハイライト内容を把握した上で会話を開始する

### 2. 会話の進め方
- ユーザーの感想・気づきを引き出す質問をする
- ユーザーが話したことに対して関連するハイライトを提示したり、他の本とのつながりを紹介したりする
- AIは自分の意見や分析を押しつけない。あくまでユーザーの思考を深めるサポートに徹する
- 会話は自然なペースで続ける。まとめを急かさない

### 3. 会話終了後の追記
ユーザーが「まとめて」「ノートに残して」などと言ったら：
- 会話の中でユーザー自身が語った言葉・気づきのみを抽出する
- AIが補足・解釈・言い換えをしない。ユーザーの言葉をそのまま使う
- 対象ノートの `## 自分の気づき` セクションに追記する（日付付きで）

#### 追記フォーマット
```markdown
## 自分の気づき

### [YYYY-MM-DD]
[ユーザーの言葉をそのまま]
```

---

## 禁止事項（Important Rules）
- `/Source` 配下のファイルは**編集・移動・削除すべて禁止**
- AI はハイライトの要約・考察・分析を**書かない**
- 「自分の気づき」「自分の発見」セクションはユーザーの言葉のみ。AI の言葉は入れない
- 新規フォルダの作成禁止（`Notes/Books` / `Notes/Themes` のみ使用）
- すべての返答・ノートは**日本語**で行う
