# Inbox Processor Skill

## トリガー条件
- **「INBOXを整形して」**：`Inbox/` 内のKeepメモをラベルに従って各フォルダへ振り分け、Inbox原本を削除する

---

## フォルダ構成

```
Inbox/                     ← KeepSidian同期後の一時置き場（整形前）
  _KeepSidianLogs/         ← KeepSidianのログ（削除禁止）
  media/                   ← 添付ファイル（削除禁止）
  [Keepのタイトル].md      ← KeepSidianが生成（ファイル名＝Keepのタイトルそのまま）

Ideas/            ← アイデア・企画・着想（Keepから・会話中・本からを問わず集約）
  アイデアリスト.md   ← 軽いアイデアのリスト
  [個別ファイル].md   ← まとまったアイデアの独立ノート
Tasks/            ← 課題・やること（Keepから・会話中・本からを問わず集約）
  課題リスト.md       ← 軽いタスクのリスト
  [個別ファイル].md   ← まとまったタスクの独立ノート
Memo/             ← 雑多メモ・気づき・単発の考察
  メモリスト.md       ← 軽いメモのリスト
  [個別ファイル].md   ← まとまったメモの独立ノート
Quotes/           ← 名言・格言
  名言集.md           ← 常にここに追記（独立ファイルは作らない）
```

---

## 実行手順

### 1. 文字化け修復（振り分けの前に必ず実行）

`Inbox/*.md` の本文が `繧/縺/繝` などの崩れた文字になることがある（KeepSidian経由の取り込みで発生しやすい）。
振り分けの前に次を実行して修復する：

```bash
python ".agents/scripts/repair_keep_mojibake.py" --apply
```

- 対象：`Inbox/*.md`（`.gitkeep` を除く）
- スクリプトが存在しない場合はスキップしてよい

### 2. 対象ファイルの検出
- `Inbox/` 内の `.md` ファイルを一覧取得する（`_KeepSidianLogs/`・`media/` フォルダは除外）
- ファイルがなければ「Inboxは空でした」と伝えて終了する

### 3. ラベルと振り分け先の決定

frontmatterの `labels` フィールドを確認し、なければ本文末尾の `#タグ` またはタイトル・本文の内容から判断する。

| Keepラベル / タグ | 振り分け先フォルダ | 内容の性質 |
|---|---|---|
| `ob-ideas` / `#idea` | `Ideas/` | ビジネスアイデア・企画・着想 |
| `ob-tasks` / `#tasks` | `Tasks/` | やること・課題・タスク |
| `ob-memo` / `#memo` | `Memo/` | 雑多なメモ・記録・気づき |
| `ob-quotes` / `#quotes` | `Quotes/` | 名言・格言 |
| ラベルなし（判断不能） | `Memo/` | デフォルトはメモ扱い |

**本文が空のファイルは振り分けせずそのまま削除する。**

**名言（`ob-quotes`）は常に `Quotes/名言集.md` の末尾に追記する（独立ファイルは作らない）。**

---

### 4. 【重要】軽いメモ vs 独立ファイルの判定

各ファイルを以下の基準で判定し、処理方法を決める。

#### ✅ リスト型追記（軽いメモ）の条件
**以下をすべて満たすもの：**
- 本文（タイトル行・空行・タグ行を除いた実質的な内容）が **3行以下**
- 見出し（`# ##`）や番号付きリスト（`1. 2.`）などの **構造化マークダウンがない**
- 短い気づき・断片的なメモ・箇条書きの羅列

→ **振り分け先リストファイルの末尾に追記する**（後述フォーマット参照）

#### ✅ 独立ファイル型の条件
**以下のいずれかを満たすもの：**
- 本文が **4行以上**
- 見出し（`# ##`）または番号付きリスト（`1. 2. 3.`）などの **構造化マークダウンがある**
- 名前付きの概念・トピックとして単体で価値を持つ内容

→ **1ファイル1メモとして振り分け先フォルダに独立したファイルを作成する**（後述フォーマット参照）

---

### 5. フォーマット詳細

#### リスト型追記フォーマット

振り分け先ファイルの末尾に以下の形式で追記する。
日付はfrontmatterの `GoogleKeepCreatedDate` から取得する（なければ今日の日付）。

```markdown
### [YYYY-MM-DD]
- [メモのタイトルまたは1行目]: [本文内容]
```

本文が複数行の場合は箇条書きで展開する：
```markdown
### [YYYY-MM-DD]
- [1行目の内容]
- [2行目の内容]
```

**整形ルール：**
- 箇条書き入力は適切な書き言葉に直す（ただし意味は変えない）
- 語順は変えない。ユーザーの表現を尊重する
- タグ（`#`）はつけない

---

#### 独立ファイル型フォーマット

振り分け先フォルダに **ファイル名 = Keepのタイトル（記号除去・整形済み）** でファイルを作成する。

**frontmatter（必須）：**
```yaml
---
date: YYYY-MM-DD
tags: [カテゴリタグ, トピックタグ1, トピックタグ2]
source: google-keep
---
```

**タグ自動付与ルール：**

カテゴリタグ（ラベルに応じて1つ付与）：
- `ob-ideas` / `#idea` → `ideas`
- `ob-tasks` / `#tasks` → `tasks`
- `ob-memo` / `#memo` → `memo`

トピックタグ（内容のキーワードから2〜4個判断して付与）：

| 内容のキーワード例 | 付与するタグ |
|---|---|
| 心理、認知、バイアス、防衛 | `心理学` |
| 自己愛、パーソナリティ、障害 | `パーソナリティ` |
| カバートアグレッション、操作 | `対人操作` |
| ビジネス、アイデア、新規事業 | `ビジネス` |
| NFCタグ、テクノロジー | `テック` |
| 記憶、認知、脳 | `認知科学` |
| 偏見、ステレオタイプ | `認知科学` |
| その他 | 内容に合わせて適宜付与 |

**本文（Keepの内容をそのまま使用）：**
- 元の構造（見出し・箇条書き・表）を保持する
- ただし `タグ行（ #memo 等）` は削除する
- 関連トピックが他の独立ファイルと重なる場合、本文末尾に `## 関連` セクションを追加して `[[ファイル名]]` 形式でウィキリンクを付与する

**ウィキリンクのつけ方：**
```markdown
## 関連
- [[カバートアグレッションの代表的な15の手法]]
- [[パーソナリティ障害に見られる防衛機能と罪悪感利用の行動]]
```

- 同じ処理バッチ内で作成した独立ファイル同士を相互リンクする
- 「カバートアグレッション」「パーソナリティ」「認知バイアス」などの共通テーマでリンクを判断する
- 同一バッチ内に関連ファイルがない場合はセクション不要

---

### 6. Inbox原本の削除
- 振り分け完了後、`Inbox/` の元ファイルを削除する
- **`_KeepSidianLogs/` フォルダは削除しない**

### 7. 完了報告
処理したファイルを以下の形式で報告する：

```
【リスト型追記】N件
- ファイル名 → Daily/Memo/メモリスト.md

【独立ファイル作成】N件
- ファイル名 → Daily/Memo/ファイル名.md

【空ファイル削除】N件
```

---

## 禁止事項
- `Source/` 配下のファイルは編集・移動・削除すべて禁止
- frontmatterに定義したタグ以外のインラインタグを勝手につけない
- すべての返答は**日本語**で行う
