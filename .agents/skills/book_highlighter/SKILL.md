# Book OCR Highlighter Skill

## トリガー条件
ユーザーが **「本の表紙の画像」** を添付したときに、このスキルが発動（発火）します。

## 実行手順（ワークフロー）
スキルが発動したら、以下の手順に従ってユーザーをサポートしてください。

### 1. 本の認識と準備
- 送られた表紙画像から「本のタイトル」と「著者名」を読み取ります。
- ユーザーに対して、「『[本のタイトル]』のハイライトを開始します。ハイライトしたいページの画像（付箋で対象箇所をマークしたもの）をアップロードしてください」と伝えます。
- 保存先ファイルのパスを決定します：`SOURCE/OCR/[本のタイトル].md` （存在しない場合は新規作成する準備をします）。

### 2. ページ画像の処理（OCRと抽出）
ユーザーがページ画像をアップロードするたびに、以下の処理を行います。
- 画像の中に **「付箋（ふせん）」** などの物理的なマーカーがあるか確認します。
- **最初の付箋から次の付箋までの間にある文章**（またはマーカーで指定されている範囲の文章）を正確に文字起こし（OCR）します。
- Kindleのハイライト機能のように、抽出したテキストをフォーマットします。

### 3. Markdownファイルへの保存
- 抽出したテキストを、指定されたMarkdownファイル (`SOURCE/OCR/[本のタイトル].md`) に追記（または作成）します。
- ユーザーのワークスペース上にディレクトリ `SOURCE/OCR/` が無い場合は作成して保存します。
- 書き込みには適切なファイル操作ツールを使用してください。

## ファイルの出力フォーマット（例）
保存するMarkdownファイルは以下のような形式でまとめます。

```markdown
# [本のタイトル]
著者: [著者名]

## ハイライト

> [付箋から付箋まで抽出された文章 1]

> [付箋から付箋まで抽出された文章 2]
```

## 注意事項（Important Rules）
- ユーザーへの返答やまとめはすべて **日本語** で行ってください。
- すべての文章を文字起こしするのではなく、意図された対象範囲（付箋と付箋の間など）のみを抽出することに集中します。
- 写真がブレていたり範囲が不明確な場合は、ユーザーに「どの部分を抽出しますか？」と確認を取ってください。
