# Book OCR Highlighter Skill

## トリガー条件
以下のいずれかでスキルが発動します。
1. ユーザーが **「OCR処理して」** または **「LINE画像をOCRして」** と依頼したとき
2. ユーザーが **本の表紙画像をチャットに直接添付** したとき

---

## 色ペン・記号のルール

色ペンがない場合は記号で同じ効果を持たせる。

| 色ペン | 記号（黒ペン代替） | 意味 | OCRタグ |
|--------|-----------------|------|---------|
| **赤** | ★ | 本文から直接得た気づき・理解・感想 | `#気づき-本文` |
| **青** | ※ | 抽象的な思考・関連性・つながり・疑問 | `#気づき-抽象` |
| **緑** | ＋ | 前提情報（著者背景・目次・登場人物・文脈） | `#前提情報` |
| マークなし | なし | 本文（全文文字起こし） | なし |

---

## 実行手順

### 1. 画像の収集

**LINEフォルダ経由の場合:**
- `LINE/images/` フォルダ内のファイルを列挙する
- `_processed_` プレフィックスがついているものはスキップ

**直接添付の場合:**
- 添付された画像を処理対象とする

---

### 2. 本のタイトル判定

以下の優先順位で決定する：
1. ユーザーのプロンプトに明記されていればそれを使う
2. 最初の画像が表紙なら表紙から読み取る
3. 本文の内容・装丁から推測する
4. 判断できなければユーザーに確認する

複数タイトルが混在している場合は内容の変化点で自動分割し、タイトルごとに別ファイルに保存する。

---

### 3. 各ページの処理

#### 全テキストをOCRに保存（必須）
色・マークに関わらず、**ページの全テキストを必ず `Source/OCR/[タイトル].md` に保存する**。
色ペンの書き込みはタグを付けてインラインで記録する。

#### ページ数の記録
各ページのテキストブロックにページ数を付与する（画像から読み取れる場合）。

#### 色ペンの書き込み処理
- **赤の書き込み文字**: 近くの本文を参照元として特定し、一緒に記録する
- **青の書き込み文字**: 近くに本文があれば参照情報として添付。余白の端など文脈から切り離されていると判断した場合は独立した思考として記録する
- **緑の書き込み文字**: 前提情報として記録する
- **書き込みなし・マークなし**: ページ全文を文字起こし

#### 切れた文の処理
複数ページにまたがる場合、ページ冒頭・末尾の文脈的に切れた文は省く。
写真がブレていて判読不能な場合はユーザーに確認する。

---

### 4. 転送処理

OCR保存後、色ペンの箇所を各転送先に以下の形式で追記する。

**赤・青（★・※）→ 内容から転送先をClaudeが判断:**

| 内容の性質 | 転送先 |
|-----------|--------|
| アイデア・応用・実践したい | `Ideas/アイデアリスト.md` |
| 理解・関連性・他の知識とのつながり | `Notes/Topics/` 該当テーマ または `Notes/Books/[タイトル].md` |
| 疑問 | `Ideas/アイデアリスト.md`（疑問ラベル付き） |
| 深い洞察・統合的な思考 | `Insights/` |
| 他者の名言・格言 | `Quotes/名言集.md` |
| 感想・共感程度 | `Source/OCR/` のみ（転送しない） |

記載形式：
```
- [書き込みテキスト]（参照: [[Source/OCR/タイトル]] p.XX）
```
本文と無関係と判断した場合は参照なしで記載。

**緑 → `Notes/Books/[タイトル].md` 冒頭の `## 前提情報` セクション:**
```
## 前提情報
- [前提情報テキスト]（p.XX）
```

---

### 5. 処理済みマーク・完了報告

- 処理済み画像は `_processed_` プレフィックスを付けてリネーム
- 完了後、以下を日本語で報告する：
  - 処理したページ数とタイトル
  - 転送先ごとの件数（気づき○件、前提情報○件）
  - 保存先ファイルパス一覧

---

## Source/OCR の出力フォーマット

```markdown
# [本のタイトル]
著者: [著者名]

## ハイライト

<!-- p.42 -->
> 本文テキスト（マークなし・全文文字起こし）

> 本文テキスト #気づき-本文
> （赤・気づき: 自分の書き込みテキスト）

> 本文テキスト #気づき-抽象
> （青・気づき: 自分の書き込みテキスト）

> 前提情報テキスト #前提情報
> （緑・前提情報: 著者の生い立ちなど）
```

---

## 注意事項
- ユーザーへの返答・報告はすべて **日本語** で行う
- `Source/OCR/` フォルダが存在しない場合は作成して保存する
- 青の書き込みが本文と関係するかどうかは、書き込み位置・内容・文脈から柔軟に判断する
- 自動リンク（他の本・Topics）は `update_vault` スキル側で処理するため、このスキルでは行わない
